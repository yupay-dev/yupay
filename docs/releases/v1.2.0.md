# Release v1.2.0: Physical Stores Dimension

**Date:** 2026-02-09
**Status:** Stable

## Overview
This release introduces the **Store Dimension**, adding physical context to sales data.

## Key Features
*   **`dim_stores`**: 50 physical store locations distributed across Peru.
*   **Customer Affinity**: Customers are assigned a `preferred_store_id` based on their city.
*   **Order Context**: Orders now include `store_id`, `pos_id` (Point of Sale), and `cashier_id`.

## Schema Changes
*   **New File**: `stores.parquet`
*   **Modified**: `orders.parquet` (added store columns), `customers.parquet` (added `preferred_store_id`).
